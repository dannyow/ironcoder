Sands of Time
by Jonathan Wight
Ironcoder 0x02 Entry (July 23rd 2006)
Description:
This is my homage to the Java based “Falling Sand” game seen all over the web.
From Wikipedia: “Falling Sand is simulation game created using Java. It features falling sand and other falling elements. It can scarcely be called a game and is more considered a sandbox. Falling Sand is a Java applet first found on the Dofi Blog via the Fark thread, later enlarged and rehosted by Chirag Mehta”
Use of Theme “Time”:
Sand of Time animates particles over time using extremely simple physics.
The initial state of the simulated world depicts an hourglass with sand particles falling through it.
Use of API “CoreGraphics”:
All graphics, including the rendering of the simulated world into an image and the loading/saving of worlds are performed using CG* apis. Much use of CGBitmapContext was made.
What went right:
The game manages to crudely simulate the particles within an hourglass. So from that criteria it is a success. I’ve also managed to include a lot of other particle types, most of which behave relatively well. Unlike many other Falling Sand games Sands of Time has a load and save function.
Performance is relatively good (although note the huge performance differences between “Debug” and “Release” builds). Good use was made of inline functions.
The code seems a lot simpler and better designed than most other Falling Sand games and will be a lot easier to hack on in the future.
What went wrong:
I am not happy with the physics code that controls gravity. It isn’t as nice as other Falling Sand games.
I didn’t get to add some of the cooler particle types (particularly plant).
Bugs
  1. •The world size is limited to 400x400 pixels. This would be very easy to fix - but I decided not to bother due to Ironcoder time limitations.
  2. •Can only save a world as a png.
  3. •Poor physics (esp. with gravity/lighter than air)
  4. •Fire don’t needs to burn a little bit more aggressively.
  5. •“Pen” routines suck. It is too hard to draw a contiguous line.
